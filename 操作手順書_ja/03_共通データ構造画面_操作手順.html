共通データ構造画面 操作手順

共通データで使用する項目の設計図を作ります。値は「共通データ管理」で入力します。

画像内の A〜D は手順番号ではなく、操作エリアを示します。
A フィールド一覧B 追加・編集・並べ替えC Json 入出力D 保存して閉じる
変更前の確認：既存項目の名前・型・階層を変えると、イベントのデータリンクや保存済みデータに影響します。先に Json 出力でバックアップしてください。

ボタンの説明

ボタン操作内容注意
フィールド追加選択位置に新しい項目を追加します。object/list を選ぶとその配下に追加されます。
フィールド編集項目名や型を変更します。「Data」は編集できません。
フィールド削除選択項目と配下の項目を削除します。イベントで使用していないか確認します。
上へ／下へ同じ階層内で項目の順番を変更します。異なる階層へは移動できません。
Json 出力現在のデータ構造をバックアップします。大きな変更の前に実行します。
Json 読込Json の構造で現在の構造を置き換えます。一致しない既存項目が見えなくなる場合があります。
保存して閉じる構造を保存し、既存データを新しい構造に合わせます。追加・編集後に必ず押します。

項目を追加する

手順 1　追加先を選びます。最上位へ追加する場合は「Data」を選びます。
手順 2　フィールド追加 を押します。
手順 3　フィールド名とデータ型を設定し、OK を押します。
手順 4　必要な項目を追加したら、保存して閉じる を押します。
完了確認：共通データ管理画面を開き、追加した項目が表示され、既存データの値が保持されていることを確認します。

型の選び方

text氏名、コード、URL などの文字。
number数量、金額などの数値。
booleanはい／いいえ、ON／OFF。
object住所など、複数項目をまとめるグループ。
list商品明細など、0件以上繰り返す項目。
安全な作業順：Json 出力 → 構造を編集 → 保存して閉じる → 共通データを確認。
