Web フロー管理
操作手順書

Web 画面の操作を「業務フロー」と「イベント」に分けて登録し、PCL データごとに自動実行するための手順書です。

対象プログラム：WebFlowManager
既定表示言語：日本語
作成日：2026年7月20日

1. このプログラムでできること

ブラウザーで繰り返し行う入力、クリック、選択、ファイルアップロードなどを登録し、順番に自動実行できます。登録した一連の操作を業務フロー、一つ一つの操作をイベントと呼びます。

業務フローを選ぶ → イベントを登録する → PCL データを準備する → 実行する

最初に覚える4つの言葉

用語意味
業務フロー「ログイン」「申込基本情報」など、目的ごとにまとめた操作単位です。
イベントページを開く、クリックする、文字を入力する、などの一操作です。
PCL データ案件名、プラン名、数量など、実行ごとに差し替えるデータです。
検出方法操作対象のボタンや入力欄をブラウザー上で見つける方法です。

2. 起動と初期設定

手順 1　PowerShell でプロジェクトフォルダーへ移動します。
cd C:\bei\git\WebFlowManager
手順 2　仮想環境を有効化し、プログラムを起動します。
..\env\Scripts\Activate.ps1
python main.py
手順 3　右上の「設定」を開き、必要に応じて言語、フォント、文字サイズ、Chrome 表示を変更します。
Chrome 表示：初めてフローを作るときや原因調査中は表示を推奨します。安定動作を確認した後は非表示でも実行できます。

3. メイン画面の見方

① 業務フロー一覧
フローの作成、順序変更、有効/無効、PCL 開始位置、実行条件を管理します。
③ 実行・共通設定
順番に実行／共通データ構造／共通 PCL データ／設定
② フロー操作
新規、名前変更、削除、上へ、下へ、PCL 開始に設定
④ イベント一覧
選択中フローの操作順、グループ、データリンク、実行条件、有効状態を表示します。
⑤ 実行ログ
現在実行中の処理、成功、失敗、スクリーンショット保存先を表示します。コピー、クリア、折りたたみが可能です。

分割線はドラッグして左右の幅を変更できます。分グループ名をクリックすると内容を展開・折りたたみできます。

4. 最短で1本のフローを作る

  1. 左側の「新規」を押し、例として ログイン と入力します。
  2. 作成したフローを選択し、右側の「イベント追加」を押します。
  3. イベント名を ログインページを開く、操作種別を goto にします。
  4. 「入力値 / URL / キー」にログインページの URL を入力して保存します。
  5. 次にユーザー名入力、パスワード入力、ログインボタンクリックを追加します。
  6. 右上の「順番に実行」を押し、ログを確認します。
おすすめ：最初は Chrome を表示し、短いフローで動作確認してください。問題が起きたイベントだけ編集すると原因を追いやすくなります。

5. 業務フローの管理

操作使い方
新規新しい業務フローを作成します。内容が分かる名前を付けます。
名前変更選択中フローの名称を変更します。
削除フローと内部イベントを削除します。必要なら先に JSON を出力してください。
上へ／下へ実行順序を一つずつ変更します。ドラッグでも並べ替えできます。
有効/無効一時的に実行対象から外します。削除せず残したい場合に使用します。
PCL開始に設定ここより前を前処理、ここ以降を PCL ごとの繰り返し処理に分けます。
実行条件PCL の値により、フロー全体を実行するか判定します。

ログインを1回だけ実行する構成

  1. 「ログイン」フローを一番上へ移動します。
  2. 次のフローを選択し、「PCL開始に設定」を押します。
  3. これでログインは前処理として1回だけ実行され、その後のフローが各 PCL に対して実行されます。
ログイン後の Cookie と localStorage は data/browser_state.json に保存され、次回実行時にも利用されます。期限切れの場合は再ログインが必要です。

6. イベントの登録

「イベント追加」を押すとイベント編集画面が開きます。無効になっている項目は、選択した操作では入力不要です。

項目入力内容
イベント名操作内容が分かる名前。例：検索ボタンを押す
有効チェックを外すと、このイベントだけ実行しません。
操作種別実行する操作を選びます。下表を参照してください。
入力値 / URL / キーURL、入力文字、キー名、待機時間、変数名、ファイルパスなどを入力します。
タイムアウト要素が見つかるまで待つ最大時間です。単位は ms、10000 は10秒です。
実行条件PCL の値により、このイベントを実行するか判断します。
失敗時の動作停止、次へ進む、ページ更新後に次へ進む、指定 URL へ移動後に次へ進む、から選びます。
検出方法／検出内容対象要素の見つけ方と値です。「現在ページから選択」を使うと入力しやすくなります。
データリンク固定入力ではなく PCL データを使う場合に指定します。

操作種別

操作用途主な入力値
gotoページを開くURL
clickボタンやリンクを押す通常は不要
fill入力欄へ文字を入力する入力文字
select選択リストから値を選ぶ選択値
wait要素が現れるまで待つ通常は不要
pressEnter、Tab などを押す例：Enter
get_text画面の文字を変数へ保存する変数名
screenshot画面を画像として保存するファイル名
pause指定時間待機する待機時間（ms）
upload_filefile input へファイルを設定するファイルパス

7. ページから要素を選ぶ・検証する

  1. イベント編集画面で「対象 URL」を確認します。
  2. 「デバッグページを開く/表示」を押します。
  3. 目的の画面まで手動で移動します。ログインや前画面の操作も手動で構いません。
  4. 「現在ページから選択」を押し、Chrome 上の対象要素をクリックします。
  5. 自動入力された検出方法と検出内容を確認します。
  6. 「現在ページで検証」を押します。対象が1件なら正常です。
途中まで自動で進める：「このイベントまで実行」を押すと、対象イベントの直前までフローを実行します。対象 Web サイトの深い画面へ毎回手動で移動する手間を減らせます。
検出数が複数の場合：同じ文字のボタンが複数ある可能性があります。role、label、placeholder を優先し、必要な場合のみ CSS や XPath を使用してください。

検出方法の目安

方法向いている対象
roleボタン、リンク、チェックボックス。例：button|保存
label項目名が付いた入力欄。
placeholder入力例が表示される入力欄。
text一意な表示文字を持つ要素。
css安定した id や class がある場合。
xpath他の方法で特定できない場合。画面変更の影響を受けやすいため注意します。

8. イベントグループ

複数イベントを一つのグループにすると、一覧が読みやすくなり、ループと失敗時リトライをグループ単位で設定できます。

  1. 「グループ追加」を押し、分かりやすいグループ名を入力します。
  2. ループする場合は「PCL データを繰り返す」を有効にし、対象リストを選びます。
  3. 失敗時に再試行する場合は「失敗時に再試行する」を有効にし、再試行回数を入力します。
  4. グループを保存します。
  5. グループを選択して「イベント追加」を押すと、グループ内へ追加できます。

既存イベントは「上へ」「下へ」で一行ずつ移動できます。グループ境界を越えると、自動的にグループ内へ入る、またはグループ外へ出る動作になります。

グループ名を1回クリックすると展開・折りたたみ、グループ名をダブルクリックすると編集です。「有効」など別の列を操作しても展開状態は変わりません。

9. 実行条件（Guard Condition）

条件を設定すると、PCL データの値に応じてフロー、グループ、イベントを実行またはスキップできます。

例： プラン.更新区分 が A の場合だけ「プラン追加」グループを実行する。
  1. 対象のフロー、グループ、またはイベントで「実行条件」を押します。
  2. 「条件を追加」を押します。
  3. PCL データパス、比較方法、比較値を入力して保存します。
  4. 複数条件では「すべて満たす（AND）」または「いずれかを満たす（OR）」を選びます。
  5. 条件一覧画面でも「保存」を押し、最後にイベントまたはグループを保存します。

比較方法には、等しい、等しくない、含む、含まない、大小比較、空、空でない、真、偽があります。

10. PCL データの準備

共通データ構造

  1. メイン画面右上の「共通データ構造」を押します。
  2. 案件名、プラン、数量など必要な項目を作成します。
  3. 繰り返すデータはリストとして作成します。
  4. 保存します。

共通 PCL データ

  1. 「共通 PCL データ」を押します。
  2. 「PCL追加」でデータを作成します。
  3. 右側の「値」列をダブルクリックして値を入力します。
  4. 「今回実行」をダブルクリックすると、実行／スキップを切り替えられます。
  5. 実行グループを設定します。同じグループ内は順番に、異なるグループは並列に実行されます。
  6. 「並列 Session 数」で同時に動かすグループ数を指定します。既定値は2です。
  7. 「現在の PCL を保存」を押します。
並列数：大きくすると速くなる場合がありますが、PC、Chrome、対象サイトへの負荷も増えます。最初は2を推奨します。

JSON と Excel

  • JSON 読込／出力：PCL データを JSON でバックアップ、復元します。
  • Excel 読込／出力：Excel で編集できます。読込は追加ではなく、Excel の内容で PCL 全体を完全復元します。

11. PCL 値・変数をイベントで使う

PCL データリンク

イベント編集画面の「構造から選択」でデータパスを選ぶと、実行時に PCL の値が入力値として使用されます。

例： 検索欄へプラン名を入力するイベントで、データリンクに プラン.プラン名 を設定します。

get_text で取得した値

  1. get_text イベントの入力値に半角英数字の変数名を入力します。例：account_id
  2. 後続イベントでは ${account_id} と記述します。

同じ実行セッション内で取得した値を、後続の URL、入力値、検出内容、失敗時の移動先 URL などに利用できます。

12. 実行する

  1. 実行対象のフローと PCL が「有効／実行」になっていることを確認します。
  2. PCL 開始位置を確認します。
  3. 必要に応じて設定から Chrome 表示を有効にします。
  4. 「順番に実行」を押します。
  5. 黄色で強調されるフローとイベント、および実行ログを確認します。

実行順序は次のとおりです。

前処理フロー（1回） → PCL 実行グループごとのフロー → 完了

13. 失敗したとき

イベントごとの失敗時動作

設定動作
実行を停止エラーを記録し、後続処理を停止します。
次のイベントへ進むエラーを記録し、そのイベントを飛ばして続行します。
ページを更新現在ページを更新してから次へ進みます。
指定 URL へ移動指定ページへ移動してから次へ進みます。

確認場所

  • 実行ログ：失敗イベント、エラー内容、保存先を確認します。
  • artifacts/：失敗時のスクリーンショットを確認します。
  • data/browser_state.json：ログイン状態の保存ファイルです。

よくある問題

症状確認方法
要素が見つからないChrome 表示を有効にし、「現在ページで検証」で検出数を確認します。画面表示完了前なら wait やタイムアウト延長を検討します。
ヘッドレス時だけ失敗する表示モードで対象要素の位置と表示状態を確認します。固定 XPath を避け、role、label、placeholder を優先します。
ログイン画面へ戻るセッション期限切れです。Chrome 表示を有効にして再ログインし、状態を保存し直します。
PCL 値が入らないデータリンクのパス、対象 PCL、リスト内の現在位置を確認します。
実行されないフロー、グループ、イベント、PCL の有効状態と実行条件を確認します。
ファイルをアップロードできない対象が input type="file" であること、ファイルパスが存在することを確認します。

14. バックアップと復元

全フロー JSON

  • 「全フロー JSON を出力」で、業務フロー、イベント、グループ、実行条件などを保存します。
  • 「全フロー JSON を読み込む」は、現在の全フローをファイル内容で置き換えます。
読込は現在の内容を置き換えます。大きな変更前、読込前、削除前には必ず JSON を出力してください。

主な保存場所

場所内容
data/flows.dbフロー、イベント、PCL、設定
data/browser_state.jsonブラウザーのログイン状態
artifacts/実行時の画像とエラー情報
settings.json操作種別、検出方法、デバッグ開始 URL

15. 実行前チェックリスト

  • 業務フローの順序は正しい
  • PCL 開始位置は正しい
  • 対象フロー、イベント、PCL が有効になっている
  • PCL の今回実行と実行グループを確認した
  • 並列 Session 数がPC環境に合っている
  • データリンクと実行条件を確認した
  • 要素の検証結果が1件になっている
  • 必要なアップロードファイルが存在する
  • 変更前の JSON バックアップを保存した
初心者向けの進め方：最初から長いフローを作らず、1～3イベントずつ追加して実行確認してください。失敗箇所が明確になり、修正が簡単です。
