# Webフロー管理

Google Chrome の操作を業務フローとして登録し、Playwright で実行するデスクトップアプリです。ページ移動、クリック、文字入力、値の取得、ファイルアップロードなどを組み合わせ、共通データごとに繰り返し実行できます。

## 主な機能

- 複数の業務フローとイベントの登録、並べ替え、有効／無効の切り替え
- `goto` / `click` / `fill` / `select` / `wait` / `press` / `get_text` / `screenshot` / `pause` / `upload_file`
- イベントグループによる階層データの反復処理
- イベントグループ単位の再試行と実行条件
- イベント失敗時の停止、続行、ページ再読み込み、指定 URL への移動
- 共通データ単位の実行／スキップと実行状態表示
- 実行グループ間の並列処理と同時 Session 数の制限
- JSON と Excel による共通データの入出力
- Chrome 表示／ヘッドレス実行
- 複数のログイン状態の保存、確認、切り替え
- ページ上の要素選択、locator 検証、対象イベント直前までのデバッグ実行
- 主 locator が失敗した場合の予備 locator 利用
- 実行中のフロー、イベント、データの状態表示
- 日本語／中国語 UI（日本語が既定）

## 動作環境

- Python 3.11 以降を推奨
- Google Chrome
- Windows PowerShell またはコマンドプロンプト（CMD）

このプログラムは Playwright の `channel="chrome"` で、端末にインストールされている Google Chrome を起動します。

## EXE 版の起動

通常利用では、`dist\WebFlowManager\WebFlowManager.exe` をダブルクリックします。
`WebFlowManager.exe` と `_internal` フォルダーは一組です。EXE だけを移動せず、`WebFlowManager` フォルダー全体を配置してください。
配布版の操作手順書は `dist\WebFlowManager\操作手順書_ja\index.html` から開けます。

## ソースコードのセットアップと起動

PowerShell とコマンドプロンプト（CMD）のどちらでも起動できます。次の方法では仮想環境を有効にする必要がありません。

```text
python -m venv env
env\Scripts\python.exe -m pip install -r requirements.txt
env\Scripts\python.exe main.py
```

仮想環境を有効にして起動する場合は、PowerShell では `.\env\Scripts\Activate.ps1`、CMD では `env\Scripts\activate.bat` を実行してから `python main.py` を実行します。

現在の実行方式では Google Chrome を使用するため、通常は `playwright install chromium` は不要です。Chrome がインストールされていない場合は、先に Chrome をインストールしてください。

## settings.json

`settings.json` で外部設定できる項目は、picker が最初に開く URL だけです。

```json
{
  "picker": {
    "start_url": "https://github.com/?locale=ja"
  }
}
```

`picker.start_url` は「デバッグページを開く/表示」やログイン状態画面で、URL が指定されていない場合に使用する既定 URL です。`http://` または `https://` で始まる URL を指定し、変更後にアプリを再起動してください。

操作種別と検出方法は実装と一体の機能であるため、外部設定ではなく `core/settings.py` の `SUPPORTED_ACTIONS` と `SUPPORTED_SELECTOR_TYPES` で固定しています。

EXE 版では、`WebFlowManager.exe` と同じフォルダーにある `settings.json` を編集します。

## 基本的な使用手順

1. 左側の「新規」で業務フローを作成します。
2. 作成したフローを選び、「イベント追加」でブラウザー操作を登録します。
3. 必要に応じて「共通データ構造」で入力項目を定義します。
4. 「共通データ」で実行するデータと値を登録します。
5. ログインが必要な場合は「ログイン状態」で認証状態を保存します。
6. 「順番に実行」を押し、「実行状況」または「実行ログ」で結果を確認します。

最初の動作確認は、「Chrome を表示」を有効にし、共通データ1件で実行すると確認しやすくなります。

## 設定メニュー

メイン画面右上の「設定」から、次の項目を変更できます。

- 外観
  - フォント
  - フォントサイズ
  - 既定の外観に戻す
- 言語
  - 日本語
  - 中国語
- Chrome を表示
- メインウィンドウを既定サイズに戻す

フォントとサイズは選択直後に反映され、次回起動時にも維持されます。「既定の外観に戻す」は Yu Gothic UI、10 pt に戻します。言語の変更は保存後、次回起動時に全画面へ反映されます。メインウィンドウの既定サイズは 1180 × 720 です。

## フローの実行順

「データ開始」に設定した業務フローより前は、前処理として1回だけ実行されます。ログインや初期ページへの移動など、各データで繰り返す必要がない処理を配置します。

「データ開始」以降は、実行対象の共通データごとに繰り返し実行されます。

```text
前処理（ログイン、初期ページへの移動など）
  ↓
実行グループ 1: PCL_001 → PCL_002
実行グループ 2: PCL_003 → PCL_004
```

同じ実行グループ内のデータは順番に実行され、異なる実行グループは並列に実行されます。「並列 Session 数」で同時実行数を1～20の範囲で設定できます。

## 共通データ（PCL データ）

共通データ管理画面では、次の操作ができます。

- データの追加、複製、名前変更、削除
- 今回実行するデータとスキップするデータの切り替え
- 実行グループの設定
- 実行状態の確認
- 階層データとリスト項目の編集
- JSON の追加読込と全データ出力
- Excel の完全復元と全データ出力

JSON 読込は既存データへ追加します。同じ名前が存在する場合は、読み込んだデータの名前が自動変更されます。

Excel 読込は追加ではありません。現在の共通データをすべて削除し、Excel 内の名前、順番、値、実行設定で完全に置き換えます。必要な現在データは、読込前に JSON または Excel で出力してください。

### データ構造

「共通データ構造」では、次の型を使用できます。

- `text`: 文字列
- `number`: 数値
- `boolean`: True／False
- `object`: 複数項目をまとめる階層
- `list`: 1件以上繰り返す項目

構造を変更して「保存して閉じる」を押すと、既存データは現在の構造に合わせて同期されます。大きく変更する前に「Json 出力」で構造をバックアップしてください。

## イベントと変数

### `get_text`

`get_text` の「入力値 / URL / キー」に、取得結果を保存する変数名を入力します。

```text
account_id
```

後続イベントでは次のように参照できます。

```text
${account_id}
```

「構造から選択」で共通データのフィールドを指定した場合は、取得結果をそのフィールドへ保存できます。

変数は同じバッチ実行中で有効です。前処理で取得した値は、各共通データの実行グループへコピーされます。

### `upload_file`

検出方法と検出内容で対象の `<input type="file">` を指定します。「入力値 / URL / キー」へファイルパスを入力するか、共通データのフィールドからパスを取得できます。

相対パスはプロジェクトフォルダーを基準に解決されます。

### `screenshot`

「入力値 / URL / キー」にファイル名を指定すると、実行中のページ全体を `artifacts/` 配下へ保存します。空の場合はイベント ID を使用したファイル名になります。

## イベントグループと再試行

「グループ追加」で複数イベントを一つのグループとして管理できます。

- 「データを繰り返す」：選択した `list` 型の共通データを順番に処理
- 「失敗時に再試行する」：グループ内で失敗した場合に指定回数まで再試行
- 「実行条件」：共通データの値に応じてグループを実行またはスキップ

データ反復と再試行は同じグループで併用できます。一覧ではグループ単位で展開、折りたたみができます。

通常イベントでは、失敗時の動作として次のいずれかを選択できます。

- 実行を停止
- 次のイベントへ進む
- ページを再読み込み
- 指定 URL へ移動

ページの再読み込みまたは URL 移動を選択した場合は、その処理が完了した後で次のイベントへ進みます。

## 実行条件（Guard Condition）

業務フロー、通常イベント、イベントグループに、共通データの値に基づく実行条件を設定できます。

- 複数条件を「すべて満たす（AND）」または「いずれかを満たす（OR）」で結合
- 等しい、等しくない、含む、含まない
- より大きい、以上、より小さい、以下
- 空、空でない
- True、False

条件を満たさないフローまたはイベントはエラーにせずスキップし、実行ログへ理由を表示します。

「データ開始」より前には現在の共通データが存在しないため、共通データを参照する実行条件は「データ開始」以降で使用してください。

## locator の登録とデバッグ

イベント編集画面では、専用のデバッグ用 Chrome を保持し、現在のページを繰り返し利用できます。

- 「デバッグページを開く/表示」：対象 URL を開く、または現在のページを前面に表示
- 「現在ページから選択」：F2 を押した後に対象要素をクリックし、locator を取得
- 「現在ページで検証」：登録中の locator で検出数を確認し、対象を強調表示
- 「このイベントまで実行」：有効な先行イベントを実行し、編集中イベントの直前で停止
- 「デバッグページを閉じる」：デバッグ用 Chrome を終了

ページから要素を選択すると、role、label、placeholder、text、CSS、XPath などから一意に検出できる方法を選びます。主 locator とともに予備 locator が保存された場合、実行時に主 locator が失敗すると予備 locator を試します。

「このイベントまで実行」で「データ開始」以降へ進む場合は、今回実行に設定されている先頭の共通データを使用します。必要な変数がある場合は、実行前に入力画面が表示されます。

## ログイン状態

メイン画面の「ログイン状態」から、サイトごとの認証状態を管理できます。

1. 「新規」で保存先を作成します。
2. URL を入力し、「認証ブラウザーを開く」を押します。
3. Chrome 上でログインや二段階認証を完了します。
4. 「現在の状態を保存」を押します。
5. 「ブラウザー状態を確認」でログイン済みか確認します。
6. 最後に画面右下の「保存」で、実行時に使用する状態を確定します。

状態名には半角英数字、下線、ハイフンを使用できます。

- `default`: `data/browser_state.json` を使用
- `none`: 保存状態を使わず、新しいブラウザーセッションを使用
- その他の名前: `data/browser_states/<状態名>.json` を使用

保存済み状態の有効期限が切れた場合は、認証ブラウザーで再ログインして保存し直してください。

## 実行結果とログ

メイン画面には次の表示タブがあります。

- 「イベント一覧」：登録イベントと現在実行中の位置
- 「実行状況」：実行中のグループ、データ、業務フロー、イベント
- 「実行ログ」：実行中のフローとイベント、完了、失敗、スキップ、再試行

実行ログは画面上でコピーまたはクリアできます。ファイルログは `log/` に保存され、各行の先頭は `2026-07-20 21:46:41` の形式です。入力値、選択値、キー、取得テキストは、ログ量と情報露出を抑えるため記録しません。

イベント失敗時のスクリーンショットと `screenshot` イベントの画像は、実行ごとに作成される `artifacts/` のフォルダーへ保存されます。

## JSON の互換性

業務フローと全フローの JSON は、現在 version 2 で出力します。version 1 と version 2 の読込に対応しています。

version 1 の `loop_start` / `loop_end`、`retry_start` / `retry_end` は読込後も利用でき、画面ではイベントグループとして表示されます。version 2 はイベントグループを入れ子構造で保存します。

全フロー JSON には業務フロー、イベント、並び順、有効状態、データ開始位置、実行条件、Chrome 表示設定が含まれます。共通データ構造と共通データは別に管理され、全フロー JSON の読込では置き換わりません。

## 保存ファイル

| パス | 内容 |
|---|---|
| `data/flows.db` | フロー、イベント、共通データ、アプリ設定 |
| `data/browser_state.json` | `default` のログイン状態 |
| `data/browser_states/*.json` | 名前を付けて保存したログイン状態 |
| `log/` | アプリの実行ログ |
| `artifacts/` | 実行時、デバッグ時のスクリーンショット |
| `settings.json` | picker が最初に開く既定 URL |
| `操作手順書_ja/` | 全体・基本・画面別の日本語操作手順と画像 |

## セキュリティ上の注意

- `data/browser_state.json` と `data/browser_states/*.json` には Cookie や localStorage などのセッション情報が含まれます。第三者へ送付したり、公開リポジトリへ登録したりしないでください。
- 共通データや実行ログに個人情報、認証情報が含まれる場合は、バックアップや共有先を制限してください。
- 並列 Session 数を増やすと、端末と対象サイトの負荷が増えます。対象サイトの利用条件に従って設定してください。
- Chrome 表示とヘッドレス実行でページのレイアウトが変わる場合は、locator を再検証してください。

## プロジェクト構成

```text
WebFlowManager/
├─ main.py                         # 起動入口
├─ app.py                          # メイン画面とアプリケーション制御
├─ i18n.py                         # 言語リソースの読み込み
├─ settings.json                   # picker の既定開始 URL
├─ core/
│  ├─ database.py                  # SQLite と JSON 入出力
│  ├─ executor.py                  # Playwright による実行処理
│  ├─ conditions.py                # 実行条件の解析と判定
│  └─ settings.py                  # 固定機能一覧と settings.json の検証
├─ ui/
│  ├─ dialogs.py                   # イベント、グループ、条件画面
│  ├─ input_data.py                # 実行変数の管理画面
│  ├─ structured_data.py           # 共通データ構造とデータ管理
│  ├─ auth_state.py                # ログイン状態管理画面
│  └─ ui_helpers.py                # 共通 UI 部品
├─ browser/
│  ├─ element_picker.py            # locator 選択、検証、デバッグ
│  └─ auth_session.py              # 認証用 Chrome の管理
├─ locales/                        # 日本語、中国語の画面文言
├─ assets/                         # アプリケーションアイコン
├─ 操作手順書_ja/                  # 日本語操作手順書一式
│  ├─ index.html                   # 全体操作手順
│  ├─ 00_基本操作手順.html
│  ├─ 01～05_*_操作手順.html       # 画面別操作手順
│  ├─ style.css
│  └─ images/                      # 手順書内の画面画像
├─ data/                           # データベースとログイン状態
├─ log/                            # アプリの実行ログ
└─ artifacts/                      # 実行結果とエラー画像
```
