# Webフロー管理

Playwright と Chrome を使用し、さまざまな Web 画面の操作を業務フローとして登録・実行するデスクトップアプリです。

## 主な機能

- 複数の業務フローとイベントの登録・並び替え
- `goto` / `click` / `fill` / `select` / `wait` / `press` / `get_text` / `pause`
- 展開可能なイベントグループによる階層 PCL データの反復処理
- イベントグループ単位の再試行、再読み込み、実行条件
- イベント失敗時のページ更新または指定 URL への移動
- PCL 単位の実行/スキップと実行状態表示
- PCL 実行グループ、グループ間並列実行、同時 Session 数制限
- JSON と Excel による PCL データの入出力
- Chrome 表示/ヘッドレス実行
- Cookie と localStorage の保存・再利用
- 実行中のフローとイベントの強調表示
- `upload_file` による file input へのファイルアップロード
- 日本語/中国語 UI（日本語が既定）

## セットアップ

Python 3.11 以降を推奨します。

```powershell
python -m venv env
.\env\Scripts\Activate.ps1
pip install -r requirements.txt
playwright install chromium
python main.py
```

Google Chrome が必要です。Playwright は `channel="chrome"` で起動します。

## 言語の切り替え

メイン画面の「言語」で次のいずれかを選択します。

- 日本語（既定）
- 中文

設定は保存され、次回起動時に反映されます。

## フローの実行順

PCL 開始より前のフローは前処理として 1 回だけ実行されます。ログイン処理はここに配置します。

PCL 開始以降のフローは、実行対象の PCL ごとに実行されます。

```text
前処理（ログインなど）
  ↓
PCL 実行グループ 1: PCL_001 → PCL_002
PCL 実行グループ 2: PCL_003 → PCL_004
```

同じグループ内では順番に実行され、異なるグループは並列に実行されます。「並列 Session 数」で同時実行数を制限できます。

## `get_text` と変数

`get_text` の「固定値」に変数名を入力します。

```text
account_id
```

後続イベントでは次のように参照できます。

```text
${account_id}
```

`upload_file` は検出方法と検出内容で対象の `<input type="file">` を指定します。入力値へファイルパスを設定するか、PCL データリンクからパスを取得できます。相対パスはプロジェクトフォルダーを基準に解決されます。

変数は同じバッチ実行中で有効です。前処理で取得した値は、各 PCL 実行グループにコピーされます。

## 再試行

「グループ追加」でイベントをまとめ、グループ編集画面の「失敗時に再試行する」を有効にして再試行回数を入力します。「PCL データを繰り返す」と同時に有効化することもでき、一覧では一つのグループとして展開・折りたたみできます。

各イベントでは、失敗時の動作として「実行を停止」「次のイベントへ進む」「ページを更新」「指定 URL へ移動」を選択できます。ページ更新または URL 移動を選択した場合は、その動作の完了後に次のイベントへ進みます。

## PCL データ

PCL データ管理画面では次の操作が可能です。

- PCL の追加、複製、名前変更、削除
- 今回実行の有効/スキップ
- 実行グループの設定
- 実行状態の確認
- JSON の入出力
- Excel の完全復元と出力

Excel 読み込みは追加ではありません。現在の PCL をすべて削除し、Excel の名前、順序、データ、実行設定を完全に復元します。

## 保存ファイル

- `data/flows.db`: フロー、イベント、PCL データ、アプリ設定
- `data/browser_state.json`: Cookie と localStorage
- `artifacts/`: スクリーンショットとエラー情報
- `settings.json`: 操作種別と検出方法の設定

## 注意事項

- 保存されたブラウザー状態にはセッション情報が含まれます。第三者と共有しないでください。
- ヘッドレスと表示モードで Web ページのレイアウトが変わる場合は、検出方法を再確認してください。
- 並列 Session 数を増やすと Chrome と対象サイトの負荷も増えます。

## locator デバッグページ

イベント編集画面では専用 Chrome を保持し、対象 Web サイトの現在ページを繰り返し利用できます。

- 「デバッグページを開く/表示」: 初回だけ開始 URL を開き、二回目以降は現在ページを前面に表示します。
- 「現在ページから選択」: ページを移動せず、現在表示中の要素から locator を取得します。
- 「現在ページで検証」: ページを移動せず、登録中の locator を強調表示します。
- 「このイベントまで実行」: 有効な前段フローを実行し、編集中イベントの直前で停止します。
- 「デバッグページを閉じる」: 保持中のデバッグ用 Chrome を終了します。

「このイベントまで実行」で PCL 開始以降へ進む場合は、実行対象になっている先頭の PCL データをデバッグ用に使用します。必要な変数がある場合は実行前に入力画面が表示されます。

## 設定メニュー

メイン画面の「設定」メニューから、次の項目を変更できます。

- 外観: 本機にインストールされている主要フォントとフォントサイズを選択します。
- 既定の外観に戻す: Segoe UI、10 pt に戻します。
- 言語: 日本語または中国語を選択します。
- Chrome を表示: 通常実行時のブラウザー表示を切り替えます。
- メインウィンドウを既定サイズに戻す: ウィンドウを 1180 x 720 に戻します。

フォントとサイズは選択直後に全画面へ反映され、次回起動時にも維持されます。言語の変更は次回起動時に全画面へ反映されます。

## Guard Condition

業務フロー、通常イベント、イベントグループへ PCL データに基づく実行条件を設定できます。条件専用の境界イベントは追加されません。

- 業務フロー一覧の「実行条件」ボタン、または条件列のダブルクリックで設定します。
- イベント編集画面の「基本設定」またはイベントグループ編集画面から設定します。
- 複数ルールは「すべて満たす（AND）」または「いずれかを満たす（OR）」で結合します。
- 等しい、等しくない、含む、大小比較、空判定、True / False 判定を利用できます。
- 条件を満たさない業務フローまたはイベントはエラーにせずスキップし、実行ログへ理由を表示します。
- 条件は全部フロー JSON の入出力対象です。条件のない旧 JSON は「常に実行」として読み込まれます。

PCL 開始位置より前の前処理には現在 PCL が存在しないため、PCL Guard Condition は設定せず、PCL 開始位置以降の業務フローで使用してください。

旧 JSON の `loop_start` / `loop_end` および `retry_start` / `retry_end` は読み込み時にそのまま利用でき、画面ではイベントグループとして表示されます。新しい JSON は入れ子のグループ形式で出力されます。
